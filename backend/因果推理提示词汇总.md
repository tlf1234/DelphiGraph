## Phase 2: 因果本体生成

### 模块：`causal_engine/ontology/causal_ontology_generator.py`

#### SYSTEM_PROMPT

```python
SYSTEM_PROMPT = """你是因果逻辑图构建专家。分析证据线索，识别核心影响因子和因果关系。

## 因子识别规则
1. 必须识别 5-8 个核心影响因子
2. 类别: macro_economic / sentiment / behavior / policy / event / other
3. 必须包含一个 "预测目标因子" 作为因果图终端节点
4. 每个因子需有明确含义和可观测性

## 因果关系规则
1. 只识别有证据支撑的关系，不凭空假设
2. 标明方向(A→B)和类型(DRIVES/INHIBITS/AMPLIFIES/TRIGGERS/CORRELATES_WITH/MITIGATES)
3. 标明强度(strong/moderate/weak)和支撑证据数量
4. 优先识别硬核事实支撑的强因果链

## 输出 JSON
{
  "factor_types": [
    { "name": "因子名", "description": "...", "category": "...",
      "measurability": "quantitative/qualitative/mixed", "examples": ["..."] }
  ],
  "causal_relations": [
    { "source_factor": "因子A", "target_factor": "因子B",
      "relation_type": "DRIVES", "strength": "strong",
      "evidence_count": 42, "reasoning": "推断依据" }
  ],
  "prediction_target": "预测目标因子名"
}

## 关键约束
- `prediction_target` 的值必须与 `factor_types` 中某个因子的 `name` 字段完全一致（字符精确匹配）
- `causal_relations` 中的 `source_factor` 和 `target_factor` 也必须与 `factor_types` 中的 `name` 完全一致"""
```

#### USER_PROMPT 构建逻辑

用户提示词由 `_build_user_prompt` 方法动态构建，包含以下部分：

```python
def _build_user_prompt(
    self,
    preprocess_result: PreprocessResult,
    task_query: str,
    additional_context: Optional[str] = None,
) -> str:
    """构建用户 Prompt"""
    parts = [f"## 预测问题\n{task_query}"]
    
    # 1. 线索统计
    parts.append(
        f"\n## 线索统计\n"
        f"- 总线索: {preprocess_result.total_signals} "
        f"| 硬核事实: {preprocess_result.hard_fact_count} "
        f"| 画像推演: {preprocess_result.persona_count}"
    )
    
    # 2. 实体索引（高频实体，按频次降序）
    # 格式：| 实体 | 类型 | 角色 | 出现频次 | 独立Agent数 |
    
    # 3. 线索聚类
    # 每个聚类包含：主题、情感倾向、锚点实体、硬核事实数、画像推演数、高质量证据样本
    
    # 4. 少数派语义簇
    # 展示少数派聚类的主题、情感、锚点实体、高质量证据
    
    # 5. 人群画像摘要（UAP v2.0）
    # 包含：职业分布、年龄段分布、地区分布、收入水平分布等
    # 各簇人群差异分析
    
    # 6. 补充信息
    if additional_context:
        parts.append(f"\n## 补充信息\n{additional_context}")
    
    return "\n".join(parts)
```

**关键提示**：
- 提示词末尾包含："注意：情感倾向由平台LLM基于汇聚数据判断，端侧Agent只提供原始数据不做预测"
- 实体索引中的 role 标注来自Agent端传感器，仅供参考，因子识别需综合判断

## Phase 1: 信号预处理

### 模块：`causal_engine/preprocessor/signal_preprocessor.py`

#### 1. 语义聚类提示词

```python
prompt = f"""请对以下线索按主题/影响因素分组。

## 预测问题
{task_query}

## 高频实体（供参考）
{entity_hint}

## 线索列表
{signals_block}

## 要求
1. 分成 3-8 个主题聚类
2. 优先以高频实体为聚类锚点
3. 为每个聚类判断其对预测问题的情感倾向（positive/negative/neutral）
4. 输出 JSON 格式

## 输出格式
{{"clusters": [
  {{"theme": "主题标签", "sentiment": "positive/negative/neutral",
   "anchor_entities": ["实体1", "实体2"],
   "signal_indices": [0, 3, 7]}}
]}}"""
```

**说明**：
- `entity_hint`: 高频实体列表，格式如 "美联储(15次), 通胀(12次), ..."
- `signals_block`: 每条线索格式为 `[索引] [硬核事实/画像推演] (相关度=0.85) 证据文本...`

#### 2. 边际贡献评分提示词

```python
prompt = f"""请为以下线索评估信息质量分数 (0-1)。

## 预测问题
{task_query}

## 线索
{signals_block}

## 评分标准
- 1.0: 罕见、具体、可验证的硬核事实
- 0.7-0.9: 有价值的具体细节
- 0.4-0.6: 一般性信息，有一定参考价值
- 0.1-0.3: 泛泛内容，缺乏具体信息
- 0.0: 重复/无关/幻觉内容

## 输出 JSON
{{"scores": [0.85, 0.40, ...]}}"""
```

**说明**：
- 每批最多评估 30 条线索
- 返回的分数数组长度必须与输入线索数量一致

## Phase 4: 因果推演

### 模块：`causal_engine/inference/causal_inference_engine.py`

#### 1. 冲突消解提示词

```python
prompt = f"""以下因果图节点存在方向冲突（同时受正向和负向因果影响）：

{conflicts_text}

请分析每个冲突，判断主导方向并给出置信度惩罚（0-0.3之间）。
输出 JSON:
{{"resolutions": [
  {{"node_id": "...", "resolved_direction": "positive/negative",
   "confidence_penalty": 0.1, "reasoning": "简要分析"}}
]}}"""
```

**说明**：
- `conflicts_text` 格式示例：
  ```
  - 节点 '市场情绪': 3条正向(权重和=1.85) vs 2条负向(权重和=1.20)
  - 节点 '政策预期': 1条正向(权重和=0.60) vs 4条负向(权重和=2.10)
  ```

#### 2. 最终结论生成提示词

```python
prompt = f"""你是宏观大脑。基于因果逻辑图分析生成预测结论。

## 预测问题
{graph.task_query}

## 因果因子（按影响力排序）
{nodes_text}

## 关键因果路径
{paths_text}

## 敏感性分析
{sensitivity_text}

## 线索统计
总: {preprocess_result.total_signals} | 硬核事实: {preprocess_result.hard_fact_count} | 画像推演: {preprocess_result.persona_count}
少数派语义簇: {len(preprocess_result.minority_clusters)} 个{minority_info}
注：方向预测由你（宏观大脑）基于因果图分析得出，端侧Agent不提供方向判断
{persona_analysis}

## 输出 JSON
{{"direction": "bullish/bearish/neutral",
  "confidence": 0.75,
  "confidence_interval": {{"low": 0.60, "mid": 0.75, "high": 0.85}},
  "key_drivers": ["因子1", "因子2"],
  "risk_factors": ["风险因子"],
  "minority_assessment": "少数派评估",
  "persona_insight": "画像维度发现（如有）",
  "one_line_conclusion": "一句话结论"}}"""
```

**说明**：
- `nodes_text` 格式示例：
  ```
  - 美联储政策 (类别=policy, 置信度=0.85, 影响力=0.92, 硬核事实=15, 画像推演=3)
  - 市场情绪 (类别=sentiment, 置信度=0.72, 影响力=0.68, 硬核事实=8, 画像推演=12, 少数派驱动)
  - 预测目标 (类别=event, 置信度=0.80, 影响力=1.00, 硬核事实=5, 画像推演=2, 预测目标)
  ```
- `paths_text` 格式示例：
  ```
  路径1: 美联储政策 → 市场流动性 → 资产价格 → 预测目标
  路径2: 经济数据 → 市场情绪 → 预测目标
  ```
- `sensitivity_text` 格式示例：
  ```
  - 美联储政策: 敏感度=0.7820
  - 市场流动性: 敏感度=0.5640
  - 经济数据: 敏感度=0.4230
  ```
- `persona_analysis` 包含人群画像分析（UAP v2.0），如职业分布、年龄段分布、各簇人群差异等

**温度参数**：`temperature=0.2`（较低温度，确保结论稳定）

## 提示词设计原则总结

### 1. 角色定位清晰
- **因果本体生成**：因果逻辑图构建专家
- **因果推演**：宏观大脑
- **未来报纸**：德尔菲未来通讯社首席记者

### 2. 输出格式严格
- 所有结构化输出都要求 JSON 格式
- 明确字段名称、类型、取值范围
- 提供输出示例

### 3. 约束条件明确
- 因子数量：5-8 个
- 聚类数量：3-8 个
- 评分范围：0-1
- 置信度惩罚：0-0.3

### 4. 证据驱动
- 强调"只识别有证据支撑的关系"
- 区分硬核事实和画像推演
- 标注证据数量和质量

### 5. 少数派关注
- 在多个阶段识别和标注少数派信号
- 要求对少数派给予特别关注
- 评估少数派的价值

### 6. 人群画像整合（UAP v2.0）
- 提供职业、年龄、地区等维度分布
- 分析不同人群的观点差异
- 将人群视角差异作为因果信号

### 7. 温度参数策略
- 结构化分析（本体、推演）：低温度 0.2-0.3
- 创意生成（报纸）：高温度 0.8
- 一般任务：中等温度 0.7

## 附录：提示词调用流程

```
原始预测数据
    ↓
Phase 1: 信号预处理 (SignalPreprocessor)
    ├─ 语义聚类提示词 → 3-8 个主题聚类
    └─ 边际贡献评分提示词 → 每条线索的质量分数 (0-1)
    ↓
Phase 2: 因果本体生成 (CausalOntologyGenerator)
    └─ 本体生成提示词 → 5-8 个因子 + 因果关系
    ↓
Phase 3: 图谱构建 (CausalGraphBuilder)
    └─ 无 LLM 调用（纯算法）
    ↓
Phase 4: 因果推演 (CausalInferenceEngine)
    ├─ 冲突消解提示词 → 解决方向冲突
    └─ 最终结论提示词 → 预测方向 + 置信度 + 关键驱动因子
    ↓
未来报纸生成 (api_service.py)
    └─ 报纸生成提示词 → 2000+ 字深度报道
```

## 注意事项

1. **字段名称一致性**：所有提示词中的字段名必须与代码中的数据模型保持一致
2. **JSON 格式要求**：所有 LLM 返回的 JSON 必须经过清理（去除 markdown 标记）和验证
3. **容错处理**：每个 LLM 调用都有 try-except 包裹，失败时有 fallback 逻辑
4. **中英文混用**：系统提示词使用中文，字段名使用英文（snake_case）
5. **端侧 vs 平台**：明确区分端侧 Agent 提供的原始数据和平台 LLM 的分析判断
