### 2. 数据模型扩展

**文件**：`backend/causal_engine/models/causal_graph.py`

**新增数据结构**：

#### ClusterNode（聚类节点）
```python
@dataclass
class ClusterNode:
    cluster_id: str
    theme: str                    # 主题标签
    sentiment: str                # positive/negative/neutral
    anchor_entities: List[str]    # 锚点实体
    signal_count: int
    hard_fact_count: int
    persona_count: int
    avg_quality_score: float      # 簇内信号平均质量
    relevance_score: float        # 与预测问题的相关度
    is_minority: bool             # 是否为少数派聚类
    persona_distribution: Dict    # 人群画像分布
    signal_ids: List[str]         # 溯源到原始信号
```

#### ClusterEdge（聚类→因子映射边）
```python
@dataclass
class ClusterEdge:
    edge_id: str
    source_cluster_id: str
    target_factor_id: str
    mapping_score: float          # 聚类主题与因子的匹配度
    signal_contribution: int      # 该聚类贡献的信号数量
```

#### CausalGraph（5层图谱）
```python
@dataclass
class CausalGraph:
    # Layer 3: 聚类层（5-15个主题聚类）
    cluster_nodes: List[ClusterNode]
    
    # Layer 4: 因子层（5-8个因果因子）
    nodes: List[CausalNode]
    
    # 边：聚类→因子映射
    cluster_edges: List[ClusterEdge]
    
    # 边：因子间因果关系
    edges: List[CausalEdge]
```

### 3. 图谱构建器升级

**文件**：`backend/causal_engine/builder/causal_graph_builder.py`

**新增步骤**：

```python
async def build(self, ontology, preprocess_result, progress_callback):
    # Step 1: 构建聚类节点（Layer 3）⭐ 新增
    graph.cluster_nodes = self._build_cluster_nodes(preprocess_result)
    
    # Step 2: 构建因子节点（Layer 4）
    graph.nodes = self._build_nodes(ontology)
    
    # Step 3: 构建聚类→因子映射边 ⭐ 新增
    graph.cluster_edges = self._build_cluster_to_factor_edges(
        graph.cluster_nodes, graph.nodes, preprocess_result
    )
    
    # Step 4: 构建因果关系边（Layer 4内部 & Layer 4→5）
    graph.edges = self._build_edges(ontology, graph.nodes)
    
    # Step 5-7: 绑定证据、计算评分、标注少数派
    ...
```

**新增方法**：

#### `_build_cluster_nodes()`
- 从预处理结果构建聚类节点
- 计算簇内信号平均质量
- 计算相关度
- 标注少数派聚类

#### `_build_cluster_to_factor_edges()`
- 为每个聚类找到最匹配的因子
- 计算映射强度（基于主题匹配度和信号质量）
- 记录因子的来源聚类
- 平均每个因子关联2-3个聚类

### 4. 编排器升级

**文件**：`backend/causal_engine/orchestrator.py`

**修改内容**：

#### `_enrich_graph_for_frontend()`
从4层改为5层图谱结构：

```python
# 原来：Agent → Signal → Factor → Target（4层）
# 现在：Agent → Signal → Cluster → Factor → Target（5层）

# 新增：
# 1. 添加 Cluster 节点（Layer 3）
for cluster_node in base["cluster_nodes"]:
    cluster_node["node_type"] = "cluster"
    enriched_nodes.append(cluster_node)

# 2. 添加 Cluster → Factor 映射边
for edge in base["cluster_edges"]:
    edge["edge_type"] = "cluster_factor"
    enriched_edges.append(edge)

# 3. 修改 Signal → Cluster 边（原来是 Signal → Factor）
cluster_id = signal_to_cluster.get(sig.signal_id)
if cluster_id:
    enriched_edges.append({
        "edge_type": "signal_cluster",
        "source": sig.signal_id,
        "target": cluster_id,
    })
```

## 🎯 边类型总结

### 5层图谱的所有边类型

| 边类型 | 连接 | 说明 |
|--------|------|------|
| `agent_signal` | Agent → Signal | Agent提交的信号 |
| `signal_cluster` | Signal → Cluster | 信号归属的聚类 ⭐ 新增 |
| `cluster_factor` | Cluster → Factor | 聚类映射到因子 ⭐ 新增 |
| `factor_factor` | Factor → Factor | 因子间因果关系 |
| `factor_target` | Factor → Target | 因子到预测目标 |

## 🔍 关键创新点

### 1. 聚类层的价值

**问题**：传统方法直接从信号跳到因子，缺乏中间层

**解决**：引入聚类层作为过渡
```
原始信号（100条）
    ↓ 语义聚类
主题聚类（7组）⭐ 新增层
    ↓ 抽象提炼
因果因子（6个）
    ↓ 因果推演
预测结论
```

**优势**：
- ✅ 完整展示数据聚合过程
- ✅ 发现被忽视的少数派主题
- ✅ 验证因子提炼的合理性
- ✅ 支持人群画像分析（UAP v2.0）

### 2. 5-15组聚类的意义

**为什么扩展数量？**
- 3-8组：主题过于粗糙，丢失细节
- 5-15组：细粒度主题，保留信息多样性

**动态调整策略**：
```python
if signal_count < 50:
    cluster_count = 5-8
elif signal_count < 150:
    cluster_count = 8-12
else:
    cluster_count = 10-15
```

### 3. 聚类→因子映射的透明化

**问题**：用户不知道因子是如何从聚类中提炼的

**解决**：显式的ClusterEdge边
```json
{
  "edge_type": "cluster_factor",
  "source": "美联储政策",
  "target": "货币政策",
  "mapping_score": 0.63,
  "signal_contribution": 25
}
```

**效果**：用户可以看到
```
"货币政策"因子 ← 
  - "美联储政策"聚类 (25条信号, score=0.63)
  - "欧央行动向"聚类 (8条信号, score=0.42)
```
