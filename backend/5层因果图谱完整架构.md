# 5层因果图谱完整架构设计

## 🎯 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                    5层因果推演图谱                            │
│                 超强因果预测逻辑展示                          │
└─────────────────────────────────────────────────────────────┘

Layer 1: Agent层 (数据源)
    │ 100+ Agents
    │ 每个Agent提交预测和证据
    ↓
Layer 2: Signal层 (原始证据)
    │ 100-500条信号
    │ 硬核事实 + 画像推演
    │ 去重、质量评分、相关度过滤
    ↓
Layer 3: Cluster层 (主题聚合) ⭐ 5-15组
    │ 语义聚类、情感标注
    │ 锚点实体、少数派识别
    │ 人群画像分布
    ↓
Layer 4: Factor层 (因果因子) 5-8个
    │ 抽象概念、可观测变量
    │ 因果关系、影响力评分
    │ 置信度计算
    ↓
Layer 5: Target层 (预测目标) 1个
    │ 最终预测方向
    │ 置信度区间
    │ 关键驱动因子
```

---

## 📊 各层详细设计

### Layer 1: Agent层（数据源）

**职责**：数据采集和初步处理

**节点类型**：Agent节点（前端展示时从原始数据提取）

**节点属性**：
```typescript
{
  agent_id: string
  agent_reputation: number
  prediction_count: number
  hard_fact_ratio: number  // 该Agent提交的硬核事实占比
}
```

**关键指标**：
- Agent数量：100-1000+
- 每个Agent提交：1-10条预测

---

### Layer 2: Signal层（原始证据）

**职责**：结构化证据存储和质量评估

**节点类型**：Signal节点（ProcessedSignal）

**节点属性**：
```typescript
{
  signal_id: string
  evidence_type: "hard_fact" | "persona_inference"
  evidence_text: string
  relevance_score: number      // 0-1
  quality_score: number         // 0-1 (边际贡献Qi)
  sentiment_tag: string         // positive/negative/neutral
  entity_tags: EntityTag[]
  cluster_id: string            // 所属聚类
  is_minority: boolean
}
```

**处理流程**：
1. 解析原始提交 → ProcessedSignal
2. 语义去重（embedding余弦相似度 > 0.92）
3. 隐私标记校验
4. 赋予证据权重（hard_fact=1.0, persona=0.1）
5. 相关度过滤（> 0.2）
6. 边际贡献评分（LLM批量评估）

**关键指标**：
- 信号数量：100-500条
- 硬核事实占比：20-40%
- 平均质量分：0.5-0.7

---

### Layer 3: Cluster层（主题聚合）⭐ 核心创新

**职责**：语义聚类和主题提炼

**节点类型**：ClusterNode

**节点属性**：
```typescript
{
  cluster_id: string
  theme: string                 // 主题标签，如"美联储政策"
  description: string
  sentiment: "positive" | "negative" | "neutral"
  anchor_entities: string[]     // 锚点实体
  
  // 统计
  signal_count: number
  hard_fact_count: number
  persona_count: number
  
  // 质量
  avg_quality_score: number     // 簇内信号平均质量
  relevance_score: number       // 与预测问题的相关度
  
  // 特殊标记
  is_minority: boolean          // 少数派聚类
  
  // 人群画像（UAP v2.0）
  persona_distribution: {
    occupation: { [key: string]: number }
    age_range: { [key: string]: number }
    region: { [key: string]: number }
    ...
  }
  
  // 溯源
  signal_ids: string[]
}
```

**聚类算法**：
- **方法**：LLM驱动的语义聚类
- **数量**：5-15组（根据线索数量和主题多样性灵活调整）
- **锚点**：优先以高频实体为聚类锚点
- **情感**：平台LLM判断簇情感倾向

**少数派识别规则**：
```python
if (
    cluster.sentiment != majority_sentiment
    and cluster.hard_fact_count > 0
    and avg_quality_score > 0.5
):
    cluster.is_minority = True
```

**关键指标**：
- 聚类数量：5-15组
- 少数派聚类：1-3组
- 平均每簇信号：10-50条

**示例聚类**：
```
聚类1: "美联储政策"
  - 情感: negative
  - 锚点实体: [美联储, 鲍威尔, 加息]
  - 信号数: 25条
  - 硬核事实: 15条
  - 少数派: false

聚类2: "市场流动性"
  - 情感: positive
  - 锚点实体: [流动性, 资金面, 货币市场]
  - 信号数: 12条
  - 硬核事实: 8条
  - 少数派: true  ⭐
```

---

### Layer 4: Factor层（因果因子）

**职责**：因果抽象和关系建模

**节点类型**：CausalNode

**节点属性**：
```typescript
{
  node_id: string
  name: string                  // 抽象因子名，如"货币政策"
  description: string
  category: "macro_economic" | "sentiment" | "behavior" | "policy" | "event" | "other"
  
  // 统计
  hard_fact_count: number
  persona_count: number
  total_evidence_count: number
  
  // 评分
  confidence: number            // 因子置信度（证据质量加权）
  impact_score: number          // 对预测目标的影响强度
  
  // 方向
  evidence_direction: "bullish" | "bearish" | "neutral"
  
  // 特殊标记
  is_prediction_target: boolean
  is_minority_driven: boolean
  
  // 溯源
  source_cluster_ids: string[]  // 来源聚类
  evidence_ids: string[]        // 原始信号
}
```

**因子识别规则**：
- 数量：5-8个核心影响因子
- 类别：macro_economic / sentiment / behavior / policy / event / other
- 必须包含1个"预测目标因子"作为终端节点
- 每个因子需有明确含义和可观测性

**因果关系类型**：
- DRIVES（驱动）
- INHIBITS（抑制）
- AMPLIFIES（放大）
- TRIGGERS（触发）
- CORRELATES_WITH（相关）
- MITIGATES（缓解）

**关键指标**：
- 因子数量：5-8个
- 平均每个因子关联：2-3个聚类
- 因果关系边：8-15条

**示例因子**：
```
因子1: "货币政策"
  - 类别: policy
  - 来源聚类: [聚类1: "美联储政策", 聚类3: "欧央行动向"]
  - 置信度: 0.85
  - 影响力: 0.92
  - 方向: bearish

因子2: "市场流动性"
  - 类别: macro_economic
  - 来源聚类: [聚类2: "市场流动性"]
  - 置信度: 0.72
  - 影响力: 0.68
  - 方向: bullish
  - 少数派驱动: true ⭐
```

---

### Layer 5: Target层（预测目标）

**职责**：最终预测结论

**节点类型**：CausalNode（is_prediction_target=true）

**预测结论**：
```typescript
{
  target_node_id: string
  direction: "bullish" | "bearish" | "neutral"
  confidence: number            // 0-1
  confidence_interval: {
    low: number
    mid: number
    high: number
  }
  key_drivers: string[]         // 关键驱动因子
  risk_factors: string[]        // 风险因子
  minority_assessment: string   // 少数派评估
  one_line_conclusion: string
}
```

### 连接2: Signal → Cluster

**边类型**：归属关系（隐式，通过cluster_id关联）

**属性**：
- cluster_id
- contribution_to_cluster_quality

---

### 连接3: Cluster → Factor（映射边）⭐ 新增

**边类型**：ClusterEdge

**属性**：
```typescript
{
  edge_id: string
  source_cluster_id: string
  target_factor_id: string
  mapping_score: number         // 聚类主题与因子的匹配度
  signal_contribution: number   // 该聚类贡献的信号数量
}
```

**映射算法**：
```python
def _build_cluster_to_factor_edges():
    for cluster in cluster_nodes:
        # 找到最匹配的因子（基于主题和锚点实体）
        best_factor = find_best_matching_node(
            cluster.theme,
            cluster.anchor_entities,
            factor_nodes
        )
        
        # 计算映射强度
        mapping_score = cluster.avg_quality_score * cluster.relevance_score
        
        # 创建映射边
        edge = ClusterEdge(
            source_cluster_id=cluster.cluster_id,
            target_factor_id=best_factor.node_id,
            mapping_score=mapping_score,
            signal_contribution=cluster.signal_count
        )
```

**关键特性**：
- 一个聚类只映射到一个最佳因子
- 一个因子可以关联多个聚类（平均2-3个）
- 映射强度反映聚类对因子的贡献度

---

### 连接4: Factor → Factor（因果边）

**边类型**：CausalEdge

**属性**：
```typescript
{
  edge_id: string
  source_node_id: string
  target_node_id: string
  relation_type: "DRIVES" | "INHIBITS" | "AMPLIFIES" | ...
  weight: number                // 0-1
  strength: "strong" | "moderate" | "weak"
  direction: "positive" | "negative" | "neutral"
  evidence_count: number
  hard_fact_ratio: number
  reasoning: string
}
```

**权重计算**：
```python
weight = min(evidence_count / 100.0, 1.0) * strength_multiplier
# strength_multiplier: strong=1.0, moderate=0.6, weak=0.3
```

---

### 连接5: Factor → Target

**边类型**：CausalEdge（同Factor→Factor）

**特殊性**：
- 所有非目标因子最终都应有路径指向目标
- 关键路径：影响力最大的因果链

   - 因子1: "货币政策" ← [聚类1]

## 🔍 关键创新点

### 1. 聚类层的价值

**传统方法**：
```
原始信号 → 因果因子 → 预测
(数据跳跃，缺乏中间层)
```

**5层架构**：
```
原始信号 → 主题聚类 → 因果因子 → 预测
(层层递进，逻辑清晰)
```

**优势**：
- ✅ 完整展示数据聚合过程
- ✅ 发现被忽视的少数派主题
- ✅ 验证因子提炼的合理性
- ✅ 支持人群画像分析（UAP v2.0）

### 2. 5-15组聚类的意义

**为什么不是3-8组？**
- 3-8组：主题过于粗糙，丢失细节
- 5-15组：细粒度主题，保留信息多样性

**动态调整策略**：
```python
if signal_count < 50:
    cluster_count = 5-8
elif signal_count < 150:
    cluster_count = 8-12
else:
    cluster_count = 10-15
```

### 3. 聚类→因子映射的透明化

**问题**：用户不知道因子是如何从聚类中提炼的

**解决**：
- 显式的ClusterEdge边
- mapping_score量化匹配度
- signal_contribution显示贡献量

**效果**：
```
用户可以看到：
"货币政策"因子 ← 
  - "美联储政策"聚类 (25条信号, score=0.63)
  - "欧央行动向"聚类 (8条信号, score=0.42)
```

## 📝 总结

5层因果图谱架构通过引入**聚类层（Layer 3）**，实现了从原始数据到抽象概念的**层层递进、逻辑清晰**的推演过程。

**核心价值**：
1. **完整溯源**：用户可以从预测结论追溯到因子、聚类、信号、Agent
2. **少数派发现**：5-15组细粒度聚类更容易发现被忽视的观点
3. **逻辑透明**：每一层的转换都有明确的算法和评分
4. **超强展示**：5层可视化展现专业的因果推理能力

这是**业界领先的因果预测架构**，远超传统的黑盒预测模型！
