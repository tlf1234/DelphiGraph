## 📝 文档维护

### 文档版本控制
- 每个文档都有版本号和更新日期
- 重大更新会在文档顶部标注
- 相关文档之间有交叉引用

### 文档更新流程
1. 修改相关文档
2. 更新版本号和日期
3. 在 [PROGRESS.md](./PROGRESS.md) 中记录更新
4. 更新本导航文档

### 文档命名规范
- 全大写：重要的顶层文档（如 README.md, SETUP.md）
- 短横线分隔：多词文档名（如 QUICK-START.md）
- 描述性命名：清晰表达文档内容

## 🤝 贡献指南

如果你想贡献文档：

1. **新增文档**：
   - 在 `doc/` 目录下创建
   - 遵循命名规范
   - 在本导航文档中添加链接

2. **更新文档**：
   - 更新版本号和日期
   - 在文档顶部说明更新内容
   - 在 [PROGRESS.md](./PROGRESS.md) 中记录

3. **文档格式**：
   - 使用 Markdown 格式
   - 添加清晰的标题层级
   - 使用代码块展示代码
   - 添加表格和列表提高可读性
