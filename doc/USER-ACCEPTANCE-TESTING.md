## 测试场景

### 场景1: 新用户注册和首次使用

**目标**: 验证新用户能够顺利注册并开始使用

**步骤**:
1. 访问DelphiGraph首页
2. 点击"登录"按钮
3. 使用Twitter账号登录
4. 查看欢迎页面和引导
5. 获取API Key
6. 浏览市场列表
7. 提交第一个预测

**预期结果**:
- 登录流程顺畅
- API Key自动生成
- 界面友好，易于理解
- 能够成功提交预测

### 场景2: 日常预测活动

**目标**: 验证用户日常使用流程

**步骤**:
1. 登录系统
2. 查看活跃市场
3. 选择感兴趣的市场
4. 查看市场详情和预测分布
5. 提交预测
6. 查看预测历史
7. 查看收益情况

**预期结果**:
- 市场信息清晰
- 预测提交成功
- 历史记录准确
- 收益计算正确

### 场景3: Python SDK使用

**目标**: 验证SDK能够正常工作

**步骤**:
1. 安装SDK: `pip install agent-oracle-sdk`
2. 配置API Key
3. 获取活跃市场
4. 提交预测
5. 查询预测历史

**预期结果**:
- SDK安装顺利
- API调用成功
- 错误处理清晰
- 文档易于理解

### 场景4: 信誉系统体验

**目标**: 验证信誉系统运作

**步骤**:
1. 查看当前信誉分和等级
2. 提交多个预测
3. 等待市场结算
4. 查看信誉分变化
5. 观察等级晋升

**预期结果**:
- 信誉分计算正确
- 等级系统清晰
- 奖励机制合理
- 进度可视化

### 场景5: 炼狱与救赎

**目标**: 验证炼狱机制（需要触发条件）

**步骤**:
1. 信誉分降至60以下
2. 进入炼狱状态
3. 查看校准任务
4. 提交校准答案
5. 完成救赎

**预期结果**:
- 炼狱触发正确
- 任务清晰明确
- 救赎机制公平
- 状态恢复正常

### 场景6: 未来模拟器

**目标**: 验证模拟器功能

**步骤**:
1. 选择一个市场
2. 生成未来模拟
3. 查看模拟内容
4. 理解共识和分歧

**预期结果**:
- 模拟生成成功
- 内容质量高
- 可视化清晰
- 速率限制有效

### 场景7: 管理员功能

**目标**: 验证管理员功能（需要管理员权限）

**步骤**:
1. 创建新市场
2. 查看待结算市场
3. 结算市场
4. 查看审计日志

**预期结果**:
- 市场创建成功
- 结算逻辑正确
- 奖励分配准确
- 日志记录完整
