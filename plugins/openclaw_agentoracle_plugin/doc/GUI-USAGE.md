# AgentOracle 插件图形界面使用指南

## 概述

AgentOracle 插件现在提供了友好的图形用户界面（GUI），让您可以更方便地管理和监控插件运行。

## 功能特性

### 1. 控制面板
- **实时状态显示**：显示插件当前运行状态（运行中/已停止）
- **一键启动/停止**：通过按钮轻松控制插件
- **配置管理**：快速访问配置设置

### 2. 运行统计
- **总任务数**：显示已处理的任务总数
- **成功/失败统计**：分别统计成功和失败的任务
- **运行时间**：显示插件已运行的时长
- **成功率**：自动计算任务成功率

### 3. 任务历史
- 显示最近 20 个处理的任务
- 包含任务 ID、标题、置信度和状态
- 按时间倒序排列，最新的在最上面

### 4. 实时日志
- 实时显示插件运行日志
- 不同级别的日志用不同颜色标识：
  - 黑色：普通信息
  - 绿色：成功消息
  - 橙色：警告
  - 红色：错误
- 支持清空和导出日志

### 5. 配置查看
- 查看当前配置信息
- API Key 自动隐藏部分字符保护隐私
- 支持重新加载配置

## 启动方式

### Windows 用户

双击运行 `start_gui.bat` 文件，或在命令行中执行：

```bash
python run.py --mode gui
```

### Linux/Mac 用户

在终端中执行：

```bash
chmod +x start_gui.sh
./start_gui.sh
```

或直接运行：

```bash
python3 run.py --mode gui
```

## 使用步骤

### 首次使用

1. **启动 GUI**
   - 双击 `start_gui.bat`（Windows）或运行 `start_gui.sh`（Linux/Mac）
   - GUI 窗口将打开

2. **检查配置**
   - 点击"配置"选项卡
   - 如果是首次运行，配置文件可能不存在
   - 点击"▶ 启动"按钮时会自动提示输入 API Key

3. **启动插件**
   - 点击"▶ 启动"按钮
   - 如果配置文件不存在，会弹出命令行窗口要求输入 API Key
   - 输入 API Key 后，插件将开始运行
   - 状态指示器变为绿色"● 运行中"

4. **监控运行**
   - 在"运行统计"面板查看实时统计数据
   - 在"任务历史"选项卡查看已处理的任务
   - 在"日志"选项卡查看详细日志

5. **停止插件**
   - 点击"⏹ 停止"按钮
   - 插件将优雅地停止
   - 状态指示器变为红色"● 已停止"

### 日常使用

1. **启动 GUI**
2. **点击"▶ 启动"**（配置已存在，无需再次输入）
3. **监控运行状态**
4. **需要时点击"⏹ 停止"**

## 常见问题

### Q: GUI 无法启动？

A: 请检查：
1. Python 是否已安装（Python 3.7+）
2. tkinter 是否可用（Python 自带，通常已安装）
3. 是否在正确的目录中运行

### Q: 点击启动后没有反应？

A: 可能是首次运行需要输入 API Key。请查看是否有命令行窗口弹出要求输入。

### Q: 如何修改配置？

A: 
1. 停止插件
2. 编辑 `config.json` 文件
3. 在 GUI 中点击"重新加载"
4. 重新启动插件

### Q: 任务历史为什么是空的？

A: 任务历史会在插件处理任务后自动更新。如果任务历史为空，可能是：
1. 插件刚启动，还没有处理任何任务
2. 数据库中没有可用的任务
3. 插件未成功连接到服务器

任务历史在 GUI 运行期间实时更新。关闭 GUI 后历史会清空。如需永久记录，请查看日志文件或导出日志。

### Q: 如何查看完整的任务 ID？

A: 在任务历史表格中，任务 ID 被截断显示。完整 ID 可以在日志选项卡中查看。

### Q: GUI 和命令行模式有什么区别？

A: 
- **GUI 模式**：提供图形界面，更直观，适合日常使用
- **命令行模式**：纯文本输出，适合服务器部署或自动化脚本

两种模式功能完全相同，只是交互方式不同。

## 技术说明

### GUI 与插件集成

GUI 现在与插件核心逻辑完全集成：

1. **实时日志显示**
   - 使用自定义 `GUILogHandler` 捕获所有日志输出
   - 日志通过线程安全的队列传递到 GUI
   - 在日志选项卡中实时显示

2. **任务历史自动更新**
   - `BackgroundDaemon` 在任务完成时触发回调
   - 任务数据自动添加到任务历史表格
   - 统计数据（成功/失败计数）自动更新

3. **线程安全设计**
   - 插件在后台线程运行，不阻塞 GUI
   - GUI 更新在主线程执行
   - 使用 `root.after()` 确保线程安全的 UI 更新
