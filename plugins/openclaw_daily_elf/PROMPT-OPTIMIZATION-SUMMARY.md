# 提示词优化总结 v1.2.0

## 优化目标

根据用户反馈，优化提示词让 OpenClaw 充分使用它的所有工具和能力来获取信息。

## 核心理念

**插件职责**：发起任务要求 → 接收结果  
**OpenClaw 职责**：使用所有工具获取信息 → 分析总结 → 生成预测

## 优化内容

### 1. 明确信息收集要求

新的提示词明确要求 OpenClaw 使用以下工具：

#### 📝 本地信息获取
- 使用**记忆检索工具**查找所有相关的历史记忆和交互记录
- 检索与任务相关的本地知识库、文档、笔记
- 查找历史对话中的相关讨论和结论

#### 👤 用户画像分析
- 分析用户的历史行为模式和偏好
- 总结用户的专业领域和兴趣方向
- 识别用户的决策风格和关注重点

#### 🌐 互联网信息检索
- 使用**网络搜索工具**查找最新的行业动态和趋势
- 搜索相关的新闻、报告、研究成果
- 获取市场数据、统计信息、专家观点

#### 📊 历史数据分析
- 检索相关的历史数据和时间序列信息
- 分析过去的趋势和模式
- 识别周期性规律和异常事件

#### 🔗 综合信息整合
- 整合所有来源的信息（本地记忆 + 用户画像 + 公网信息 + 历史数据）
- 交叉验证不同来源的信息
- 识别信息的可靠性和时效性

### 2. 要求信息来源总结

新的输出格式要求 OpenClaw 列出使用的信息来源：

```
### 📋 信息来源总结
- 本地记忆：[列出使用的记忆和知识]
- 用户画像：[总结相关的用户特征]
- 公网信息：[列出搜索到的关键信息来源]
- 历史数据：[说明使用的历史数据]
```

这样可以：
- ✅ 验证 OpenClaw 确实使用了工具
- ✅ 了解信息的来源和可靠性
- ✅ 评估分析的全面性

### 3. 强化主动性

添加了明确的提示：

```
**重要提示**: 请充分使用你的所有工具和能力，不要局限于已有知识。
主动搜索、检索、分析，提供最全面和准确的预测。
```

```python
message = f"""【智能预测任务】

## 任务描述
{task_description}

## 分析要求
1. **信息收集**: 使用你的记忆检索工具查找相关的本地记忆和信息
2. **趋势分析**: 识别关键趋势和模式
3. **风险评估**: 评估潜在风险和不确定性
4. **预测结论**: 给出明确的预测结论和置信度
5. **行动建议**: 提供可执行的行动建议
```

**问题**：
- ❌ 只提到"记忆检索工具"，不够全面
- ❌ 没有明确要求使用网络搜索、用户画像等
- ❌ 没有要求列出信息来源
- ❌ 不够主动

```python
4. **预测结论**: 给出明确的预测结论和置信度（基于信息质量和数量）
5. **行动建议**: 提供可执行的、个性化的行动建议
```

- ✅ 要求列出信息来源
- ✅ 强化主动性

## 预期效果

当 OpenClaw 收到优化后的提示词时，它会：

1. **理解任务范围**：不仅仅是回答问题，而是全面收集信息
2. **主动使用工具**：
   - 调用记忆检索工具
   - 调用网络搜索工具
   - 分析用户画像
   - 查询历史数据
3. **整合信息**：交叉验证不同来源的信息
4. **生成报告**：包含信息来源、分析、预测、建议

## 边界原则

这次优化进一步强化了边界原则：

**我们不做的事**：
- ❌ 不直接读取 OpenClaw 的内部文件
- ❌ 不访问 `~/.openclaw/memory/` 目录
- ❌ 不预先收集信息

**我们做的事**：
- ✅ 发送详细的任务要求
- ✅ 明确指示 OpenClaw 使用工具
- ✅ 接收 OpenClaw 的分析结果
- ✅ 保存结果到本地

## 总结

v1.2.0 的提示词优化实现了：

1. **全面性**：覆盖所有 OpenClaw 能接触到的信息源
2. **主动性**：明确要求主动搜索和检索
3. **可追溯性**：要求列出信息来源
4. **个性化**：要求分析用户画像
5. **边界清晰**：插件发起任务，OpenClaw 使用工具
