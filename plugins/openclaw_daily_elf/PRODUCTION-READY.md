## ⚠️ 注意事项

### 1. Gateway 必须运行
确保 OpenClaw Gateway 正在运行：
```bash
wsl -d Ubuntu-24.04 bash -c "systemctl --user status openclaw-gateway"
```

### 2. Token 配置正确
确认 `GATEWAY_TOKEN` 与 Gateway 配置一致。

### 3. 网络连接稳定
虽然有自动重连，但稳定的网络环境仍然重要。

### 4. 超时时间合理
根据任务复杂度调整 `TIMEOUT` 值。

## 🔍 故障排查

### 问题 1: 连接失败
**症状**: `🔌 WebSocket 错误`

**解决**:
1. 检查 Gateway 是否运行
2. 验证 URL 和 Token
3. 查看 Gateway 日志

### 问题 2: 超时
**症状**: `⏰ 总超时 (300秒)`

**解决**:
1. 增加 `TIMEOUT` 值
2. 简化任务描述
3. 检查 AI 模型状态

### 问题 3: 重试失败
**症状**: `❌ 达到最大重试次数`

**解决**:
1. 增加 `MAX_RETRIES`
2. 检查网络稳定性
3. 重启 Gateway
